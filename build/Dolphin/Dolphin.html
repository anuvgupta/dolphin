bool set(string $table, string $child, array $data = null)

Modify or create a child within a table

This method sets column values for a child row (specified by ID) within a table.
If the table does not exist, it is created. If the child row does not exist within the table, it is created.
If any one column does not exist within the table, it is added to the table (the type can be specified).


Uses of set

Use 1: Setting child data in a table with columns
Each key in this associative array represents a table column, and each value is the value to be set in that column for the given child's row.

$data_example_1 = [
     'username'   =>  'joesmith123',
     'firstname'  =>  'Joe',
     'lastname'   =>  'Smith',
     'desc'       =>  'Hi! I like long walks on the beach.',
     'age'        =>  26
];
$dolphin->set('users', 'd6h45', $data_example_1);

If the columns exist, the child's columns are updated accordingly.

However, in some cases, the columns specified may not exist. If a column does not exist, the table is first ALTERed, adding the column with type varchar(255).
You can specify the type of a column in case it does not already exist and you want it to have a different type than varchar(255).

Use 2: Setting child data in a table while creating new columns with different data types
Each key in this associative array represents a table column, and each value is another associative array.
      The value mapped to val is the value to be set in that column for the given child's row.
      The value mapped to type is the MySQL type of the column, and if the column does not exist in the table, it will be created with this type.

$data_example_2 = [
     'username'  => [
         'val'   => 'joesmith123',
         'type'  => 'varchar(100)' // perhaps I want usernames to be restricted to a shorter length
     ],
     'firstname' => 'Joe', // no specified type, because the default type is `varchar(255)`, or I know this column already exists
     'lastname'  => [
         'val'   => 'Smith',
         'type'  => 'varchar(255)' // this is unnecessary, because the default type is `varchar(255)`
     ],
     'desc'      => [
         'val'   => 'Hi! I like long walks on the beach.',
         'type'  => 'text'
     ],
     'age'       => [
         'val'   => 26,
         'type'  => 'integer'
     ],
];
$dolphin->set('users', 'd6h45', $data_example_2);


Use 3: Creating an empty child row
Example 3: in some cases, the data parameter is null, or simply not passed (in which case it defaults to null). In this case, if the child row does not exist, it is created. If it does exist, nothing happens. In future versions of Dolphin, this may change so that if the child row does exist, all of its attributes (columns) are cleared.

$dolphin->set('users', 'd6h45', null);
$dolphin->set('users', 'd6h45'); // the same thing

Parameters

string $table Name of the table to be modified. If the table does not exist, it will be CREATEd.
string $child ID of the child row of values to be set/modified. If the child does not exist, it will be INSERTed rather than UPDATEd.
array $data Technically optional. An associative array containing attributes of the child to set/modify, and their corresponding values (as well as their corresponding types, if necessary). See the above method description for details on how to format.

Return Value

bool Indicates success of the operation. If any one database operation fails, the method immediately returns false and logs the error; in this case, use Dolphin::error to get the exact error.

mixed push(string $table, array $data = null, integer $length = 10)

Add a new child with a unique ID to a table

This method creates a new child row with a unique ID within a table, setting column values for it as well.
The ID of the child created is random, alphanumeric, and unique within the table; in other words, the new child's ID will be different from the IDs of every other child in the table.
This method uses Dolphin::set, so if the table does not exist, it is created; and if any one column does not exist within the table, it is added to the table (the type can be specified).

Parameters

string $table Name of the table to be modified. If the table does not exist, it will be CREATEd.
array $data Technically optional. An associative array containing attributes of the new child, and their corresponding values (as well as their corresponding types, if necessary).
See the description of Dolphin::set for a detailed explanation on how to format the $data array.
integer $length Optional. Specifies the length of the generated ID. Defaults to 10 characters.

Return Value

mixed The generated alphanumeric string ID of the newly created child; also indicates success of the operation.
If any one database operation fails, the method immediately returns bool false (instead of returning the ID) and logs the error; in this case, use Dolphin::error to get the exact error.

Get data from a table based on child or query

This method retrieves data from a table. It can retrieve all the data in the table, or the data for a specific child row, or the data of all child rows matching a query.


Uses of get

Use 1: Get table data
Get all the data from a table. There are three options here - get an array the child IDs, get all the child data, or get specific columns from every child.

mixed error(integer|bool $num = 0)

Get logged errors

Returns most recent error, or specified error, or number of errors, depending on the parameter provided.


Uses of error

Use 1: Get most recent error
Call the method with no arguments to get the most recent error as a string. The method will return false if there are no logged errors.

$last_error = $dolphin->error();
if ($last_error) echo "The last error: $last_error";
else echo "No errors";


Use 2: Get a specific error
Pass the integer ID of the desired error (the most recently error logged has an ID of 0, and each previous error has an incremented ID) to get that error as a string. Returns false if the error is not found.

$error_6 = $dolphin->error(6);
if ($error_6) echo "The sixth error: $error_6";
else echo "Error 6 does not exist";


Use 3: Get total number of errors
Pass true to get the total number of errors. Useful for calculating ID of specific errors.

$num_errors = $dolphin->error(true);
echo "Dolphin has logged $num_errors error(s)" . PHP_EOL;
if ($num_errors > 1) echo "The second-to-last error: " . $dolphin->error($num_errors - 2);
else echo "Dolphin has only logged one error";

Parameters

integer|bool $num Technically optional. Indicates integer ID of error to retrieve (leave blank for most recent error), or bool true to retrieve total number of errors logged.

Return Value

mixed Total integer number of errors, or error string matching the given ID (or of the most recent error), or bool false if error requested is not found.
